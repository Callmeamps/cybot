## Register

product

## Users

TBD. Primary use case: interacting with an LLM agent in a 3D spatial interface. The user is likely technical, comfortable with API keys and model selection, and drawn to spatial/experiential UI over plain chat windows.

## Product Purpose

A 3D spatial interface for conversing with AI agents. The agent lives in a navigable Three.js environment instead of a flat chat panel. Themes, characters, and tool cards give the interface personality and context. The goal is to make AI interaction feel spatial, alive, and fun rather than transactional.

## Brand Personality

Spatial, playful, cyberpunk. The interface should feel like a holographic terminal in a stylized void: glowing geometry, floating elements, dark backgrounds with accent color pops. Personality comes from the 3D scene, not from decorative UI chrome.

## Anti-references

- Not a generic ChatGPT-style chat wrapper
- Not corporate, sterile, or plain
- Not overdesigned with unnecessary chrome, gradients, or decorative borders
- Not a SaaS landing page pretending to be an app

## Design Principles

1. **Spatial first.** The 3D scene is the interface, not a backdrop. UI elements float in the space, they don't sit on top of it.
2. **Personality through geometry.** Character comes from shapes, motion, and light, not from illustrations or icons.
3. **Dark by default.** The void is the aesthetic. Light themes exist but feel like an alternate world, not a safety default.
4. **Functional over decorative.** Every glow, every animation, every element earns its place. No ornament.
5. **Performance is a feature.** The scene must run smooth on low-end hardware. If it lags, the magic breaks.

## Accessibility & Inclusion

Reduced motion support is built in. Additional accessibility features (high contrast mode, screen reader improvements, keyboard navigation) are planned for a settings menu. No specific WCAG target yet; pragmatic improvements as the product matures.
